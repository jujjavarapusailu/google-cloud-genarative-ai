Backend API Integration

Overview

This activity focused on developing the backend infrastructure of the EduGenie Learning Assistant using the FastAPI framework. RESTful API endpoints were created for every educational feature, allowing the frontend interface to communicate efficiently with the AI-powered backend modules.

FastAPI was selected because of its high performance, asynchronous request handling, simplicity, automatic API documentation, and seamless integration with AI services. The backend architecture was designed using a modular approach, where each endpoint was linked to its corresponding AI functionality.

Implemented API Modules

Separate RESTful API routes were developed for each educational feature. Every endpoint receives user input, validates the request, executes the respective AI module, and returns structured JSON responses to the frontend.

API Endpoint Function
/qa Handles educational question answering.
/explain Generates simplified explanations for educational topics.
/quiz Creates AI-generated multiple-choice quizzes.
/summarize Summarizes lengthy educational content.
/learn/recommendations Generates personalized learning paths and recommendations.

Backend Workflow

  • Receives user requests through query parameters or JSON payloads.
  • Validates the received input data.
  • Invokes the corresponding AI module for processing.
  • Generates AI-powered educational responses.
  • Returns structured JSON responses to the frontend.
  • Implements error handling for missing or invalid input values.

Outcome

The backend API integration established smooth communication between the frontend interface and AI-powered educational modules. This architecture enables real-time educational assistance, efficient response generation, and seamless interaction between users and the EduGenie Learning Assistant, providing a scalable foundation for future enhancements.
